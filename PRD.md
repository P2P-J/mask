## 0. 이번 버전에서 확정된 것 (v0.2 → v0.3 변경점)

사용자 결정으로 **범위가 크게 좁아지고 명확해짐:**

1. **Windows 11 전용.** Windows 10 미지원. → 가상캠을 `MFCreateVirtualCamera`(Win11 전용 API)로 단순화.
2. **PC 웹캠 전용.** 폰을 카메라로 쓰는 모드(YYCam식 폰 브리지) 없음. macOS 없음.
3. **자체 네이티브 가상 카메라 확정.** OBS 영구 의존 아님. 앱 켜면 OBS/디스코드/줌 장치 목록에 "Mask"가 바로 뜨는 YYCam식 구동.
4. **전부 무료 오픈소스로만 제작. 비용 0원.** 상용 뷰티 SDK(FaceUnity·Banuba 등) 전면 배제. 유료 코드서명 인증서도 사지 않음.
5. **목표 품질 = YYCam Pro 수준의 자연스러운 보정.** FabSoften 등 공개 학술 기법으로 자체 구현.
6. **삭제:** 텔레메트리, 자연스러움 정량 검증 KPI, 수익화 논의 — 현 단계 범위 밖.

## 1. TL;DR

방송용 뷰티캠 시장(YYCam Pro, NVIDIA Broadcast, XSplit VCam 등)은 **구독료·플라스틱 보정·RTX 종속·중국산 데이터 우려** 중 하나 이상의 약점을 가진다. Mask는 **100% 로컬·무료·오픈소스 스택**으로, "내 얼굴이 이 PC를 떠나지 않는" 프라이버시와 "티 안 나는 자연스러움"을 핵심 차별점으로 한다.

벤치마크는 **YYCam Pro**다. YYCam의 구동 방식(앱 켜면 자체 가상캠으로 잡힘)과 보정 기능(스무딩·피부보정·리쉐이프)을 목표로 하되, YYCam이 의존하는 상용 SDK 대신 **MediaPipe + 자체 WebGL2 셰이더 + FabSoften식 주파수 분리**로 무료 재현한다. 가상캠은 Windows 11의 `MFCreateVirtualCamera`(VCamSample 레퍼런스)로 직접 구현한다.

---

## 2. 배경 및 문제

- 라이브 방송에서 외모 보정은 사실상 표준. 스트리머의 자신감·시청 유지율과 직결.
- 기존 솔루션의 약점: **(a) 구독료**, **(b) 플라스틱 얼굴**, **(c) RTX 종속**, **(d) 중국산 앱의 데이터 신뢰 문제(YYCam은 JOYY 계열)**.
- 핵심 기술(실시간 얼굴 추적·보정)이 무료 오픈소스(MediaPipe, Apache 2.0)로 상업 이용까지 가능해져 인디 개발자도 진입 가능.

**Mask가 푸는 문제:** 한국 스트리머가 *돈 안 내고*, *RTX 없이*, *내 얼굴 데이터를 외부에 안 보내고*, *플라스틱 안 되게* 보정하고 싶다.

## 3. 목표와 차별화

### 3.1 제품 목표
- **G1.** 일반 노트북 내장 GPU(iGPU)에서 1080p·30fps 이상으로 자연스러운 실시간 보정.
- **G2.** 설치 후 5분 내 방송 연동 — **자체 가상캠**으로 OBS/치지직/SOOP에 바로 잡히게.
- **G3.** "보정한 티가 안 난다"는 인상(질감 보존 보정).

### 3.2 차별화 (우선순위 재정렬)
1. **🔒 "이 PC를 떠나지 않는다" — 100% 로컬 프라이버시.** 중국산 YYCam 대비 한국 시장의 진짜 무기. 마케팅 1면.
2. **🎯 티 안 나는 자연스러움.** FabSoften 주파수 분리 + 시간축 안정화로 *기술적으로* 뒷받침.
3. **🇰🇷 치지직·SOOP 5분 연동.** 한국 플랫폼 특화 온보딩 + 자체 가상캠.

> "무료·가벼움"은 차별점이 아니라 **진입 조건(table stakes)**으로 강등. 진짜 방어선은 위 3가지.

### 3.3 비목표 (Non-goals)
- 폰을 카메라로 쓰는 모드(YYCam식 폰 브리지)
- Windows 10 / macOS / 모바일
- 전신/체형 보정, 가상 메이크업 풀세트(e-커머스용)
- 상용 SDK 라이선스, 유료 코드서명
- 텔레메트리/사용량 수집

## 5. 경쟁 분석 요약 (리서치 검증 반영)

| 제품 | 구동/포지션 | 강점 | 약점 | Mask의 대응 |
|---|---|---|---|---|
| **YYCam Pro** (SoDo Live, JOYY계열) | 폰캠+자체 가상캠 브리지, 프리미엄 | 풍부한 뷰티/리쉐이프, 멀티플랫폼, **자체 가상캠 자동구동** | 구독, 중국산 데이터 우려 | 가상캠 구동·보정 깊이 벤치마크, 무료·로컬로 차별화 |
| NVIDIA Broadcast | GPU AI 효과 | 고품질 배경/노이즈 | **RTX 필수**, 뷰티 약함 | 비RTX·뷰티 특화 |
| XSplit VCam | 배경 제거 | 간편 | 얼굴 보정 약함 | 얼굴 보정 본업 |
| FaceUnity/Banuba (SDK) | 상용 보정 엔진 | 25포인트 리쉐이프, 데스크톱(Electron) 지원 | **유료(과금)** | MediaPipe+자체 셰이더로 무료 대체 |

**검증된 핵심 사실:**
- YYCam은 **자기 자신을 Windows 가상 카메라로 등록** → OBS 드롭다운에서 선택 시 클라이언트 자동 실행(표준 호스트-스폰 패턴). **이게 우리가 복제할 구동 모델.**
- YYCam 보정 기능: 스무딩, 피부 보정, 좌우대칭/비율/작은얼굴/팔자주름 리쉐이프, 뷰티필터, 가상배경. (출력 품질은 독립 벤치마크 없음 — 우리가 자연스러움으로 이길 여지)

## 6. 기능 요구사항 (P0=MVP / P1=V1 / P2=차후)

### 6.1 영상 입출력
- **[P0]** 웹캠 장치 선택 + 실시간 프리뷰
- **[P0]** 해상도/fps 설정 (720p/1080p, 30/60fps)
- **[P0]** 미러(좌우반전) 토글, 줌/크롭/프레이밍, 16:9 종횡비 — *기본기, v0.2에서 누락분 추가*
- **[P0]** **자체 가상 카메라 출력 (`MFCreateVirtualCamera`)** — OBS/디스코드/줌 장치 목록에 "Mask"로 노출. *v0.2의 P2에서 P0로 격상(핵심 가치)*
- **[P1]** 가상캠 자동 구동(장치 선택 시 앱 자동 실행 — YYCam식 호스트-스폰)

### 6.2 얼굴 추적
- **[P0]** MediaPipe Face Landmarker, GPU delegate, LIVE_STREAM 모드, 468 랜드마크
- **[P0]** **랜드마크 시간축 안정화 (One Euro Filter)** — 깜빡임/헤엄 방지. *v0.2 누락 핵심 추가*
- **[P0]** 얼굴 미검출 시 **원본 패스스루**(보정 미적용)로 안전 동작
- **[P1]** CNN 기반 안정화(Steadiface식)로 고도화
- **[P2]** 다중 인물(2인)

### 6.3 보정 기능 (핵심)
- **[P0] 피부 스무딩 — FabSoften식.** 랜드마크+**YCbCr 스킨톤 결합 마스크**에만 적용(눈·입·눈썹·머리카락·안경·손 제외). ADF(피부확률 기반 가변 Guided Filter) + **웨이브렛 주파수 분리(고주파 HH 텍스처 보존)**로 플라스틱 방지. 강도 슬라이더. *마스크 정밀화·주파수 분리를 P0로 끌어올림*
- **[P0] 얼굴 자동 노출/밝기.** 얼굴 영역 기반 노출 보정 — *조명이 스무딩보다 체감 큼, v0.2 누락분 P0 추가*
- **[P0] 밝기/대비/톤** 기본 색보정 슬라이더
- **[P1] 잡티 제거** (Canny 검출 + 컨실, FabSoften 전처리)
- **[P1] 화이트밸런스 / 따뜻함(warmth)**
- **[P1] 배경 블러/교체** (MediaPipe Selfie Segmentation)
- **[P2] 페이스 리쉐이프** (랜드마크 메시 워프 — 갸름/눈크게/작은얼굴/팔자주름, 강도 상한). **WebGPU 전환 대상.**
- **[P2] 치아/눈 화이트닝, 메이크업(립/블러셔)** (TPS 워프 디커플링 렌더)

### 6.4 프리셋 / UX
- **[P0]** 보정 강도 실시간 슬라이더
- **[P0]** 보정 On/Off 토글 + Before/After 비교
- **[P0]** **패닉 단축키** — 한 번에 즉시 원본 패스스루(신뢰 장치). *v0.2 누락 추가*
- **[P1]** 프리셋 저장/불러오기 (자연스러움/방송용/회의용)
- **[P1]** 단축키 커스터마이즈

### 6.5 설정
- **[P0]** 입력 장치/해상도/fps 설정 저장
- **[P1]** 성능 모드(품질 vs 속도) + 적응형 품질(FPS 미달 시 자동 하향)

## 8. 기술 아키텍처 & 스택

### 8.1 파이프라인
```
웹캠 캡처(getUserMedia)
  → 얼굴 추적(MediaPipe, 저해상도)
  → One Euro 랜드마크 안정화
  → 스킨 마스크(랜드마크 + YCbCr)
  → WebGL2 셰이더 보정(FabSoften 스무딩 + 자동노출 + 색보정)
  → 캔버스 출력
  → [네이티브 브리지] 공유메모리
  → MFCreateVirtualCamera → OBS/디스코드/줌
```

### 8.2 스택 (전부 무료·상업 이용 가능)
| 레이어 | 기술 | 라이선스 | 비고 |
|---|---|---|---|
| UI/언어 | TypeScript + React (한국어 i18n) | MIT | 슬라이더 패널 |
| 캡처 | `getUserMedia` / MediaStream | — | |
| 얼굴 추적 | `@mediapipe/tasks-vision` Face Landmarker | Apache 2.0 | GPU delegate, 468점 |
| 안정화 | One Euro Filter (자체 구현) | — | 깜빡임 제거 |
| 세그멘테이션 | MediaPipe Selfie Segmentation | Apache 2.0 | 배경(P1) |
| GPU 렌더 | **WebGL2 (MVP) → WebGPU (리쉐이프부터)** | 자체 셰이더 | 아래 8.3 참고 |
| 보조 처리 | OpenCV.js (필요 시) | Apache 2.0 | 웨이브렛 등 |
| 빌드 | Vite | MIT | |
| 가상캠 | **`MFCreateVirtualCamera`** (VCamSample 레퍼런스, Win11) | MIT 계열 | 네이티브 C++ 모듈 |
| 패키징 | Electron | MIT | Win11 빌드/설치 |

### 8.3 WebGL2 먼저, WebGPU는 리쉐이프부터 (v0.2 내부모순 해소)
- **MVP = WebGL2.** 스무딩·주파수 분리·색보정·자동노출은 블러 패스라 WebGL2로 60fps 충분. iGPU/드라이버 호환성도 WebGL2가 안정적.
- **WebGPU = P2 페이스 리쉐이프(메시 워프)부터.** 컴퓨트 이점이 실제로 필요한 무거운 워프에만 도입. (구형 환경 WebGL2 폴백 유지)
- *근거: 저사양 호환이 핵심 가치인데 가장 불안정한 기술(WebGPU)을 MVP에 넣는 건 자해. v0.2의 "WebGPU 주력" → 철회.*

### 8.4 가상 카메라 — 확정 경로 (리서치 검증)
- **`MFCreateVirtualCamera` (Windows 11 전용).** `smourier/VCamSample`(MIT)이 동작하는 레퍼런스. **유저모드 COM DLL + 미디어 소스** 구조라 **커널 드라이버/WHQL 인증 불필요** → 무료 제작 가능.
  - *주의: 완전체 "Frame Server Custom Media Source"는 WHQL 필요하지만, 우리는 그 경로 안 씀.*
- Electron 앱 ↔ 네이티브 가상캠 모듈을 **공유메모리로 프레임 전달**.
- 앱 켜면 장치 목록에 "Mask" 노출, 선택 시 호스트 프로세스 자동 구동(YYCam식). → **G2 "5분 연동" 실현.**
- **알려진 한계(비용 0 정책 결과):** 배포 시 유료 코드서명(Authenticode/EV) 없으면 SmartScreen 경고가 뜰 수 있음. 개발/로컬은 문제없음. **이 경고는 감수**(돈 안 씀). 추후 무료 서명 옵션만 검토.

### 8.5 Build only — 상용 SDK 안 씀
- **전 단계 Build.** FaceUnity·Banuba 등 상용 SDK는 과금 모델(특히 MAU) 때문에 **영구 배제**. 품질이 한계에 부딪혀도 자체 셰이더·공개 학술 기법으로만 해결.

### 8.6 자연스러움 알고리즘 노트 (리서치로 확보한 청사진)
- **스무딩은 반드시 스킨 마스크에만** — 랜드마크 마스크(눈/입/눈썹 루프 제외) + **YCbCr 스킨톤 마스크 결합**(머리카락·안경·손 거르기).
- **FabSoften 주파수 분리:** 미세 텍스처는 고주파 **HH 밴드**에 존재 → LL/HL/LH는 스무딩본 복사, **HH는 원본↔스무딩본을 텍스처 강도(TI)로 블렌딩**(`α_HH = 0.5·TI + 1.0`, `TI = Σ|I−I_bilateral|·P_skin/255`). → "매끈하지만 플라스틱 아님"의 핵심.
- **ADF:** Guided Filter의 `r`·`ε`를 피부 확률 마스크 기반으로 윈도마다 가변.
- 모든 처리 GPU 셰이더 — CPU bilateral은 프레임 드랍 원인.

### 8.7 성능 최적화 전략
1. 워프 단일 패스 합성(변위 맵 누적 후 1회 샘플링)
2. 메시 기반 워프(468점 토폴로지, GPU 래스터라이저 보간)
3. 추적/렌더 디커플링(랜드마크 격프레임+보간, 렌더 60fps)
4. 저해상도 추적 / 원본 해상도 보정·출력
5. GPU 상주 파이프라인(CPU↔GPU 복사 최소화, 텍스처 풀 재사용)
6. 경량 엣지보존 블러(dual/Kawase로 bilateral 근사)
7. 고성능 GPU 강제(`powerPreference:'high-performance'`, Electron 외장 GPU 강제)
8. 적응형 품질(FPS 측정 → 해상도·워프 정밀도 자동 하향, 하드 비활성 대신 그레이스풀 다운그레이드)

## 9. 단계별 로드맵 (리스크 순으로 재배치)

### Spike 0 — 3대 핵심 리스크 동시 검증 · ~1.5주
가장 무서운 가정부터 깬다. 셋 다 통과해야 제품이 성립.

1. **성능 스파이크:** Vite+TS 스켈레톤 → `getUserMedia` → MediaPipe Face Landmarker(GPU) → 메시 오버레이 → **FPS/지연/GPU메모리 HUD 내장.** 완료: iGPU 실기기 1080p 30fps↑.
2. **가상캠 스파이크(최대 리스크):** `MFCreateVirtualCamera`(VCamSample 기반)로 임의 프레임을 가상캠으로 출력. 완료: OBS/줌/디스코드 장치 목록에 "Mask" 노출 + 컬러바 보임.
3. **안정화 스파이크:** One Euro Filter로 랜드마크 떨림 제거 시각 확인. 완료: 정지 시 메시가 안 떨림.

**리스크:** iGPU 성능 미달 → 다운스케일 추적. 가상캠 서명 이슈 → 로컬 동작 우선 검증.

### Phase 1 — MVP: 자연스러운 스무딩 + 자체 가상캠 · ~3주
- **목표:** 보정 영상이 자체 가상캠으로 방송에 뜬다.
- 작업: 스킨 마스크(랜드마크+YCbCr) → FabSoften WebGL2 스무딩 → 얼굴 자동노출 + 밝기/대비 → 강도 슬라이더/On-Off/Before-After/**패닉 단축키** → MFCreateVirtualCamera 출력 → 미러/줌/크롭 → 설정 저장 → Electron Win11 패키징.
- **완료:** 설치 파일로 배포, OBS 없이 "Mask" 가상캠 송출, 자연스러운 강도 조절.

### Phase 2 — V1: 기능 확장 · ~3~4주
- 가상캠 자동 구동(호스트-스폰), 잡티 제거, 화이트밸런스/따뜻함, 배경 블러/교체, 프리셋, 적응형 품질, 워치독.
- **완료:** 장시간 안정, 프리셋 동작, 한국 플랫폼(치지직/SOOP) 온보딩 가이드.

### Phase 3 — V2: 차별화 + 최적화 · 추후
- 페이스 리쉐이프(WebGPU 메시 워프, 강도 상한), 화이트닝/메이크업(TPS 디커플링), 다중 인물, CNN 안정화.
- **완료:** 리쉐이프 자연스러움 확보, 저사양 60fps 근접.
