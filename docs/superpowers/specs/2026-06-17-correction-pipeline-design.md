## 1. 목적

MVP UI 셸(피치 파스텔, 라이브 프리뷰)을 **OBS식 도크 레이아웃**으로 재구성하고, Canvas 2D 렌더러를 **WebGL2 셰이더 파이프라인**으로 교체해 **실제 보정(슬라이더가 영상에 적용)** 을 구현한다. 스트리머 타깃이 이미 OBS에 익숙하다는 점을 활용한 친숙한 UI.

이번 v1 범위: **풀 FabSoften 피부 스무딩 + 색보정 + 레이어/장면 구조**. 리쉐이프·치아 화이트닝은 후속 사이클.

## 2. 범위

### 포함 (In Scope)
- **OBS식 레이아웃:** 상단 큰 미리보기 + 하단 도크 4개 — 장면 / 레이어(소스목록) / 편집(믹서 자리) / 제어. 피치 파스텔 유지.
- **WebGL2 렌더 파이프라인:** 비디오 텍스처 → 셰이더 패스 체인(FBO 핑퐁) → 캔버스. 메시 오버레이는 위에 겹친 별도 2D 캔버스.
- **풀 FabSoften 피부 스무딩:** 스킨 마스크(랜드마크 폴리곤 + YCbCr) + 엣지보존 블러(Kawase) + 웨이브렛식 주파수 분리(고주파 텍스처 보존). 슬라이더: 강도, 질감 보존.
- **색보정 레이어:** 밝기/대비/톤/화이트밸런스(단일 셰이더).
- **보정 상태 모델:** 레이어(enabled+params) 단일 출처, UI·렌더러 공유.
- **장면(Scene):** 전체 보정값 스냅샷. 추가/전환/저장/삭제, localStorage 영속(설정값만 로컬).
- **fps 버그 수정:** 선택 fps(30/60)가 실제 반영되도록.
- **캔버스 찢김(보류 #8):** WebGL2 전환 과정에서 해소 확인.

### 제외 (Out of Scope) — 후속 사이클
페이스 리쉐이프(머리통/갸름), 치아 화이트닝, 잡티 제거, ADF 정밀화, 배경 블러, One Euro 안정화, 네이티브 가상캠, React 전환, Electron, 장면 이름변경/드래그 순서변경, 레이어 순서 드래그.

## 3. 아키텍처 결정

- **바닐라 TS 유지.** 상태 복잡도가 커지지만 v1까지는 가벼운 자체 상태 스토어로 충분. React는 후속.
- **보정 상태 모델이 단일 출처.** UI는 상태를 읽고 쓰고, 렌더러는 매 프레임 상태를 읽어 유니폼 설정. UI↔렌더 직접 결합 없음.
- **레이어 순서 고정(v1).** 보정 순서는 결과에 영향(스무딩→색보정). v1은 의미 있는 고정 순서, on/off만. 드래그 재정렬은 후속.
- **메시 오버레이 분리.** GL 캔버스(보정 영상) 위에 투명 2D 캔버스(메시). 파이프라인과 디커플링.

## 4. 모듈 구조

기존 재사용: `camera.ts`(변경 없음), `tracker.ts`(랜드마크 제공, 변경 없음), `metrics.ts`(변경 없음).
교체: `renderer.ts`(Canvas 2D → WebGL2). `ui/controls.ts`·`ui/panel.ts`·`index.html`·`styles.css`는 OBS 레이아웃으로 재구성.

신규(제안 경계):
- `src/state/store.ts` — 보정 상태 모델 + 순수 reducer(레이어 토글, param 변경, 장면 전환/저장). **단위테스트 대상.**
- `src/state/persist.ts` — localStorage 직렬화/복원.
- `src/state/defaults.ts` — 기본 카테고리/레이어/장면 정의.
- `src/gl/pipeline.ts` — WebGL2 컨텍스트, FBO 핑퐁, 패스 체인 실행, 비디오 텍스처 업로드.
- `src/gl/passes/skinMask.ts` — 랜드마크 폴리곤 + YCbCr 스킨 마스크 패스.
- `src/gl/passes/blur.ts` — Kawase 엣지보존 블러 패스.
- `src/gl/passes/smoothing.ts` — 주파수 분리 합성(스무딩 최종).
- `src/gl/passes/color.ts` — 색보정 패스.
- `src/gl/shaders/*.glsl(.ts)` — 셰이더 소스.
- `src/gl/skinMaskMath.ts` — YCbCr 판정·α 계산 등 순수 헬퍼. **단위테스트 대상.**
- `src/ui/scenes.ts` — 장면 도크. `src/ui/layers.ts` — 레이어 도크. `src/ui/editor.ts` — 편집 도크. `src/ui/dockControls.ts` — 제어 도크 + 카메라/해상도/fps + 진단.
- `src/main.ts` — rAF 루프: tracker.detect → store 읽기 → pipeline.render(video, landmarks, state) → 메시 오버레이 → 진단.

> 파일 경계는 계획 단계에서 확정. 순수 로직(store reducer, skinMaskMath)만 단위테스트, 셰이더/렌더는 Windows 수동 검증.

## 6. FabSoften 스무딩 패스 (PRD §8.6)

1. **스킨 마스크** = 랜드마크 얼굴 폴리곤(눈·입·눈썹 루프 제외) ∩ YCbCr 스킨톤 → 스킨 확률 `P_skin`.
2. **엣지보존 블러** = Kawase/dual-filter 다중 패스로 bilateral 근사(`I_bilateral`).
3. **주파수 분리 합성** = 고주파 `HF = I − I_bilateral`; 결과 `= I_bilateral + α·HF`, `α = 0.5·TI + 1.0`, `TI = |I − I_bilateral|·P_skin`. 최종 = 스킨 마스크·강도로 원본↔스무딩 블렌드.
- 슬라이더: **강도**(블렌드 0~1), **질감 보존**(α 스케일).
- 전부 GPU 셰이더(CPU bilateral 금지 — 프레임 드랍).

## 7. 색보정 패스

단일 셰이더: 밝기(+/−), 대비, 톤(웜/쿨), 화이트밸런스. 슬라이더 0~100 → 유니폼. 얼굴 무관 전체 적용(자동노출은 후속).

## 8. fps 버그

증상: 60fps 선택해도 좌상단 FPS 30. 원인 후보: (a) 웹캠이 1080p에서 30fps만 제공, (b) rAF 루프 throttle이 `actualFps`로 제한. 처리: 카메라 실제 제공 fps 진단 → 선택 fps를 카메라가 지원하면 그 값으로 throttle, 미지원이면 실제값을 진단에 명확 표기(거짓 60 방지). 셀렉터는 카메라 능력에 맞게.

## 9. 에러 처리

- WebGL2 미지원/컨텍스트 생성 실패 → 명확한 에러, 크래시 없음(추후 2D 폴백 검토는 후속).
- 셰이더 컴파일 실패 → 콘솔+에러 표시.
- 얼굴 미검출 → 스킨 마스크 빈 값, 보정 미적용(원본 패스스루), 루프 지속.
- localStorage 없음/손상 → 기본 장면으로 복구.

## 10. 테스트

- 단위테스트: `state/store.ts` reducer(토글/param/장면전환/저장 불변성), `gl/skinMaskMath.ts`(YCbCr 판정, α 계산), `ui/format.ts`(기존).
- 수동(Windows): 슬라이더가 실제 영상에 반영, 스무딩 자연스러움(플라스틱 아님), 색보정 반영, 장면 전환 시 보정 전체 변경·새로고침 후 복원, 메시 정합, fps 정확, 찢김 해소, 1080p 성능 유지(추론 9ms 기준 헤드룸).

## 11. 완료 기준

1. OBS식 4도크 레이아웃이 피치 파스텔로 보인다.
2. 미리보기에 **실제 FabSoften 스무딩이 적용**되고 강도/질감보존 슬라이더가 즉시 반영된다(자연스러움).
3. 색보정 슬라이더가 반영된다.
4. 레이어 on/off가 동작한다.
5. 장면 추가/전환/저장 + 새로고침 후 복원이 동작한다.
6. fps 선택이 실제와 일치한다.
7. 캔버스 찢김이 사라졌다.
8. 1080p에서 실시간 성능 유지(프레임 드랍 없음).
