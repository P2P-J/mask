## 1. 목적

YYCam Pro의 구성을 데스크톱 가로 화면에 맞춰 푼 **MVP UI 셸**을 만들고, 거기에 **현재 동작하는 카메라+얼굴추적 파이프라인을 라이브로 연동**한다. 스타일은 **피치 파스텔 + TDS(Toss Design System) 원칙**(화이트 베이스·강한 위계·둥근 모서리·옅은 그림자)으로 깔끔·세련하게.

이 단계의 산출물은 "보기 좋은 껍데기 + 진짜 움직이는 프리뷰"다. 실제 피부 보정(FabSoften WebGL2 셰이더)은 **이 작업 범위 밖**(후속 MVP 단계).

## 2. 범위

### 포함 (In Scope)
- 피치 파스텔 / TDS 스타일의 전체 앱 레이아웃 (상단바·좌측 레일·중앙 프리뷰·우측 패널·하단 CTA).
- **중앙 프리뷰에 실제 라이브 영상 연동** — 기존 `camera.ts`+`tracker.ts`+`renderer.ts` 재사용. 미리보기는 **무조건 항상 표시**되는 중앙 주역.
- **동작하는 컨트롤:** 카메라 선택 / 해상도(720p·1080p) / fps(30·60) 셀렉터, 메시 오버레이 On/Off, 진단(FPS/추론/프레임/해상도/얼굴검출) 패널.
- **플레이스홀더 컨트롤(UI만, 실제 효과 없음):** 좌측 레일(얼굴 활성 + 몸매·필터·화장·배경 흐림), 우측 탭(피부·색보정 활성 / 윤곽·배경 흐림)과 슬라이더(드래그 시 숫자만 변동), 하단 CTA(가상캠 시작·보정 On/Off·Before/After·패닉) — 시각적 상태 토글까지만, 하부 기능 미연결은 "준비 중"으로 표기.

### 제외 (Out of Scope)
실제 피부 스무딩/색보정/자동노출 셰이더, 스킨 마스크, One Euro 안정화, 네이티브 가상캠(MFCreateVirtualCamera), React 도입, Electron 패키징, 프리셋 저장. **전부 후속 작업 단위.**

## 3. 아키텍처 결정

- **바닐라 TypeScript 유지.** 지금 동작하는 PoC 모듈(`camera/tracker/renderer/metrics`)을 그대로 재사용한다. React는 슬라이더 상태·프리셋 등 상태 복잡도가 실제로 커지는 **실 보정 단계에서 도입**(Spike 0 설계의 기존 방침과 일치).
- **기존 PoC를 새 셸로 승계.** 별도 throwaway 목업 파일을 만들지 않는다. `index.html`을 새 레이아웃으로 교체하고, 기존 측정용 HUD는 **접이식 진단 패널**로 이동시켜 Spike 0 파트 A의 FPS 측정 능력을 보존한다.
- **피치 파스텔은 CSS 변수(디자인 토큰)로** 정의해 한 곳에서 관리(TDS식 토큰화).

## 4. 모듈 구조

기존(재사용, 거의 변경 없음):
- `src/camera.ts` — 장치 열거 / getUserMedia / 스트림. **변경 없음.**
- `src/tracker.ts` — MediaPipe FaceLandmarker. **변경 없음.**
- `src/renderer.ts` — canvas 비디오 + 메시 오버레이. **변경 없음.**
- `src/metrics.ts` — FPS/지연 측정. **변경 없음.** (단위테스트 유지)

신규/개편:
- `index.html` — 새 레이아웃 마크업(상단바·레일·프리뷰 캔버스·우측 패널·하단 CTA·진단 패널).
- `src/styles.css` — 피치 파스텔 디자인 토큰 + TDS식 컴포넌트 스타일(Button/ListRow/Tab/슬라이더/Badge 느낌).
- `src/ui/controls.ts` — **동작 컨트롤** 배선: 카메라/해상도/fps 셀렉터, 오버레이 토글, 진단 패널 업데이트. (기존 `hud.ts` 발전형 — `hud.ts`는 이 모듈로 흡수/대체)
- `src/ui/panel.ts` — **레일 + 탭/슬라이더 패널** 렌더·상태. 현재는 플레이스홀더 상태(활성/흐림, 슬라이더 값 표시)만.
- `src/main.ts` — 오케스트레이션(rAF 루프). 기존 구조 유지, 메트릭을 진단 패널로 보내고 컨트롤 상태를 읽도록 소폭 수정.

> 단위테스트는 순수 로직(`metrics.ts`)만 유지. UI 모듈은 Windows 실기기 수동 검증.

## 5. 레이아웃 / 데이터 흐름

```
┌─ 상단바: ● Mask | 카메라▾ · 1080p · 30fps · [진단▾] ─────────┐
├──────┬─────────────────────────────────┬───────────────────┤
│ 레일 │  미리보기 (라이브, 필수)          │  얼굴 보정         │
│ [얼굴]│  ← <canvas>: 비디오 + 메시오버레이 │  [피부][색보정]    │
│ 몸매 │  좌상단 ● LIVE 30fps 뱃지         │  윤곽·배경(흐림)   │
│ 필터 │                                  │  스무딩 ──●─ 45    │
│ 화장 │                                  │  질감보존 ──●  70  │
│ 배경 │                                  │  잡티  ●──  20     │
│  +   │                                  │  자동노출 [ON]     │
├──────┴─────────────────────────────────┴───────────────────┤
│ ● 가상캠 시작(준비중)  보정 On/Off  Before/After  패닉        │
└─────────────────────────────────────────────────────────────┘
```

데이터 흐름(기존 유지):
```
camera(MediaStream → <video>)
  → main rAF 루프
    → tracker.detect(video) → {faces, inferenceMs}
    → renderer.draw(canvas, video, faces, overlay)   ← 중앙 프리뷰
    → metrics → controls.updateDiagnostics(snapshot)  ← 접이식 진단 패널
  controls: 카메라/해상도/fps 변경 → camera 재시작
  panel: 탭/슬라이더/레일 = 플레이스홀더 상태만
```

## 6. 비주얼 스펙 (피치 파스텔 / TDS)

- 포인트: `#ee9678`(primary) / `#f6b9a3`(secondary) / 슬라이더 트랙 `#f7e3da`
- 베이스: 크림 화이트 `#fffcfa`, 패널 `#fff`, 레일 `#fdf1ec`, 보더 `#f7e7e0`
- 텍스트: 제목 `#5e463d`, 본문 `#8a7068`
- 라운드: 카드/패널 12–16px, 버튼/탭/슬라이더 999px(pill)
- 그림자: 옅게 `0 8px 24px rgba(232,137,107,.12)`
- 흐림(준비 중) 요소: opacity .5, 점선 보더 placeholder
- 한국어 UI, 시스템 폰트 스택.

## 7. 상태 / 상호작용

- **탭 전환:** 클릭 시 활성 탭 + 슬라이더 세트 교체(피부 ↔ 색보정). 흐린 탭(윤곽·배경)은 비활성.
- **슬라이더 드래그:** 우측 숫자 실시간 변동(0–100). 실제 영상 효과는 아직 없음.
- **레일:** 얼굴만 활성, 나머지 흐림(클릭 시 "준비 중" 안내).
- **하단 CTA:** 가상캠 시작 = 비활성("준비 중"). 보정 On/Off·Before/After·패닉 = 시각 토글만(하부 보정 미구현이라 효과 없음, 의도된 자리표시).
- **진단 패널:** 상단바의 [진단] 토글로 펼침. FPS/추론/프레임/요청·실제 해상도/얼굴검출/JS힙 표시 + 메시 오버레이 체크박스. (Spike 0 파트 A 측정 보존)

## 8. 에러 처리 (기존 수준 유지)

- 카메라 권한 거부 → 프리뷰 영역에 안내 메시지, 크래시 없음.
- 얼굴 미검출 → 진단 패널 "없음" 표시, 오버레이 스킵, 루프 지속.
- MediaPipe 로드 실패 → 명확한 에러 메시지.

## 9. 테스트

- `metrics.ts` 단위테스트 유지(회귀 안전망).
- 나머지는 Windows 실기기 수동 검증: 라이브 프리뷰 표시, 셀렉터 동작, 메시 오버레이 정합, 탭/슬라이더 인터랙션, 진단 수치.

## 10. 완료 기준

1. 피치 파스텔/TDS 스타일의 전체 레이아웃이 보인다.
2. 중앙 프리뷰에 실제 웹캠 라이브 영상 + 메시 오버레이가 뜬다.
3. 카메라/해상도/fps 셀렉터, 오버레이 토글, 진단 패널이 실제 동작한다.
4. 레일·탭·슬라이더·하단 CTA가 의도된 위계로 보이고 시각 인터랙션이 된다(하부 기능 미연결은 "준비 중" 명시).
5. Spike 0 파트 A FPS 측정이 진단 패널로 여전히 가능하다.

## 11. 후속

- 실제 FabSoften WebGL2 피부 스무딩 → 슬라이더 실연결.
- One Euro 안정화(파트 C), 네이티브 가상캠(파트 B), React 전환, Electron 패키징.
